# 自动化组件生成脚本 (add.js) 详解

## 一、脚本概述

`add.js` 是一个基于 Node.js 的**命令行自动化工具**，用于快速创建统一结构的 React 组件。它通过以下核心技术实现：

- **文件系统操作** (`fs` 模块)：创建目录、读写文件
- **路径处理** (`path` 模块)：跨平台路径拼接
- **模式匹配** (`glob` 模块)：批量扫描模板文件
- **模板引擎** (`handlebars` 模块)：变量替换与动态生成

---

## 二、代码逐段详解

### 2.1 模块导入与 ES Module 环境适配

```javascript
import path from 'path';
import fs from 'fs';
import { glob } from 'glob';
import handlebars from 'handlebars';
import { fileURLToPath } from 'url';

const __filename = fileURLToPath(import.meta.url);
const __dirname = path.dirname(__filename);
```

**功能说明：**

| 代码 | 作用 |
|------|------|
| `import path from 'path'` | 引入 path 模块，用于拼接和解析路径（兼容不同操作系统） |
| `import fs from 'fs'` | 引入文件系统模块，用于文件的读写和目录创建 |
| `import { glob } from 'glob'` | 引入 glob 模块，用于匹配文件路径（如 `*.hbs`） |
| `import handlebars from 'handlebars'` | 引入 Handlebars 模板引擎，用于变量替换 |
| `import { fileURLToPath } from 'url'` | ES Modules 中获取当前文件路径的工具 |

**为什么需要 `__dirname`？**

由于脚本使用了 ES Modules（`import`/`export`），而在 ES Module 中没有内置的 `__dirname` 变量。因此需要通过 `import.meta.url`（当前文件的 URL）转换为文件路径来手动构造 `__dirname`。

```javascript
// ES Module 中获取 __dirname 的标准方式
const __filename = fileURLToPath(import.meta.url);  // 当前文件的绝对路径
const __dirname = path.dirname(__filename);         // 当前文件所在目录
```

### 2.2 命名转换函数

```javascript
const varCase = str => {
  return str.replace(/-/g, '_').replace(/^./, char => char.toUpperCase());
}

const lowCase = str => {
  return str.replace(/-/g, '_').replace(/^./, char => char.toLowerCase());
}
```

**功能说明：**

这两个函数用于将命令行输入的组件名（如 `search-bar`）转换为符合 JavaScript/React 规范的命名格式：

| 函数 | 输入示例 | 输出示例 | 用途 |
|------|---------|---------|------|
| `varCase` | `search-bar` | `SearchBar` | PascalCase，用于组件名/类名 |
| `lowCase` | `search-bar` | `searchBar` | camelCase，用于文件夹名/变量名 |

**转换逻辑详解：**

```javascript
// 1. replace(/-/g, '_')：全局替换所有连字符为下划线
//    "search-bar" -> "search_bar"

// 2. replace(/^./, char => char.toUpperCase())：匹配并大写首字符
//    "search_bar" -> "Search_bar"
```

### 2.3 主执行逻辑（IIFE 立即执行函数）

```javascript
(async () => {
  // ... 核心逻辑
})();
```

使用 **IIFE（立即调用函数表达式）** 的原因：
- 允许使用 `async/await` 语法
- 避免污染全局作用域
- 自启动执行，无需外部调用

---

### 2.3.1 第一步：获取参数与创建目录

```javascript
const component = process.argv[2];         // 获取命令行第3个参数（组件名）
const dirName = lowCase(component);         // 转换为 folderName 格式
const componentName = varCase(component);   // 转换为 ComponentName 格式
const targetDir = path.join(process.cwd(), 'src', dirName);  // 目标路径

await fs.promises.mkdir(targetDir, { recursive: true });
```

**功能说明：**

| 代码 | 说明 |
|------|------|
| `process.argv[2]` | Node.js 命令行参数数组，`argv[0]` 是 node 路径，`argv[1]` 是脚本路径，`argv[2]` 才是用户输入的组件名 |
| `process.cwd()` | 获取当前终端工作目录（项目根目录） |
| `path.join(...)` | 跨平台路径拼接，生成如 `D:\project\src\button` 的路径 |
| `fs.promises.mkdir(..., { recursive: true })` | 创建目录，`recursive: true` 表示自动创建父级目录，目录已存在也不报错 |

**执行示例：**

```bash
node add.js search-bar
# component = "search-bar"
# dirName = "searchBar"
# componentName = "SearchBar"
# targetDir = "D:\project\src\searchBar"
```

### 2.3.2 第二步：扫描模板目录

```javascript
const tplDir = path.join(__dirname, 'template').replace(/\\/g, '/');
const tplFiles = glob.sync(`${tplDir}/*.hbs`);
```

**功能说明：**

| 代码 | 说明 |
|------|------|
| `__dirname` | 脚本所在目录（`scripts/`） |
| `path.join(__dirname, 'template')` | 拼接为 `scripts/template/` |
| `.replace(/\\/g, '/')` | 将 Windows 反斜杠 `\` 替换为正斜杠 `/`（glob 库在 Windows 下需要） |
| `glob.sync('*.hbs')` | 同步获取所有 `.hbs` 文件的路径数组 |

**glob 模式说明：**

- `*` 匹配任意字符（不包含 `/`）
- `*.hbs` 匹配当前目录下所有 `.hbs` 文件
- `**/*.hbs` 可匹配子目录中的文件

---

### 2.3.3 第三步：循环处理模板并生成文件

```javascript
for (const filePath of tplFiles) {
  const content = await fs.promises.readFile(filePath, 'utf-8');  // 读取模板内容
  const template = handlebars.compile(content);                   // 编译模板
  const result = template({ dirName, componentName });            // 注入变量

  const fileName = path.basename(filePath)  // 获取文件名（不含路径）
    .replace('components', dirName)          // 替换文件中的 "components"
    .replace('.hbs', '');                     // 移除 .hbs 后缀
  const newPath = path.join(targetDir, fileName);  // 新文件路径

  await fs.promises.writeFile(newPath, result);  // 写入文件
  console.log(`write ${newPath} success`);       // 打印成功日志
}
```

**处理流程图：**

```
template/index.tsx.hbs  →  读取内容  →  Handlebars 编译  →  变量替换  →  输出 index.tsx
                            ↓
                      替换 "components" 为 "searchBar"
                      替换 {{componentName}} 为 "SearchBar"
                      替换 {{dirName}} 为 "searchBar"
```

**文件名转换示例：**

| 模板文件名 | 转换后文件名 |
|-----------|-------------|
| `index.tsx.hbs` | `index.tsx` |
| `index.test.tsx.hbs` | `index.test.tsx` |
| `index.scss.hbs` | `index.scss` |

## 三、Handlebars 模板变量

在模板文件（`.hbs`）中，可以使用以下变量：

| 变量名 | 来源 | 示例值 |
|--------|------|--------|
| `{{componentName}}` | `varCase()` | `SearchBar` |
| `{{dirName}}` | `lowCase()` | `searchBar` |

**模板示例：**

```handlebars
// index.tsx.hbs
import React from 'react';
import styles from './{{dirName}}.module.scss';

interface {{componentName}}Props {
  children?: React.ReactNode;
}

const {{componentName}} = ({ children }: {{componentName}}Props) => {
  return <div className={styles.root}>{children}</div>;
};

export default {{componentName}};
```

**编译结果：**

```tsx
import React from 'react';
import styles from './searchBar.module.scss';

interface SearchBarProps {
  children?: React.ReactNode;
}

const SearchBar = ({ children }: SearchBarProps) => {
  return <div className={styles.root}>{children}</div>;
};

export default SearchBar;
```

## 四、使用方法

### 4.1 命令行调用

```bash
node scripts/add.js <component-name>
```

### 4.2 示例

```bash
# 创建单个单词组件
node scripts/add.js button
# 输出目录：src/button

# 创建多单词组件（使用连字符分隔）
node scripts/add.js search-bar
# 输出目录：src/searchBar

# 创建长名称组件
node scripts/add.js data-table
# 输出目录：src/dataTable
```

### 4.3 生成的文件结构

假设 `scripts/template/` 目录下有以下模板文件：

```
template/
├── index.tsx.hbs          →  src/<component>/index.tsx
├── index.scss.hbs         →  src/<component>/index.scss
├── index.test.tsx.hbs     →  src/<component>/index.test.tsx
└── index.stories.tsx.hbs  →  src/<component>/index.stories.tsx
```

## 六、注意事项

### 6.1 命名规范

- 组件名建议使用**小写加连字符**（如 `my-component`）
- 脚本会自动转换为适合代码的格式
- 避免使用特殊字符或空格

### 6.2 Windows 兼容性

- 脚本使用 `path.join()` 确保跨平台路径拼接
- 使用 `.replace(/\\/g, '/')` 解决 glob 在 Windows 下的路径问题

### 6.3 模板目录结构

- 所有模板文件必须放在 `scripts/template/` 目录下
- 模板文件必须以 `.hbs` 为扩展名
- 支持嵌套目录结构（通过 glob 递归匹配）
